# Project Portfolio Page - Wang Pengjin

## Project: SudoCook
**SudoCook** is a Java-based Command-Line Interface (CLI) application designed to help users manage recipes and kitchen
inventory efficiently. It enables students and home cooks to track their ingredients and discover what they can cook
through an intuitive text interface.

### 2. Enhancements Implemented

* **`delete-r` command** (`DeleteRecipeCommand.java`)
    * Removes a recipe from `RecipeBook` by 1-based index.
    * Non-numeric indices are rejected at parse time; out-of-range indices are caught via `IndexOutOfBoundsException` inside `RecipeBook.removeRecipe()`.

* **`recommend-r` command suite** (`RecommendByIngredientCommand`, `RecommendByInventoryCommand`, `RecommendByMissingCommand`)
    * Three recommendation modes under one command prefix:
        * **Ingredient-based** (`n/INGREDIENT_NAME`): recommends recipes whose required quantity for the named ingredient does not exceed current stock.
        * **Inventory-based** (no flag): recommends recipes that can be fully prepared with current inventory using `canMake()`.
        * **Missing-based** (`missing/N`): recommends recipes missing at most N ingredients, reporting the exact shortfall per item.
    * All modes use case-insensitive matching and `≤` quantity comparison.
    * All modes support unit converting, avoiding unit incompatibility within the same unit family.
    * Comprehensive unit tests covering boundary cases, empty inventory, shortfall display, and parser validation.
